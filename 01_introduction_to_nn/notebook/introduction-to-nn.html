Customer Churn Prediction¶

Applications: telecom service providers, banks, employers¶

Business Problem: We have a dataset of a bank with 10,000 customers. The dataset contains lots of attributes of the customers measured over last 6 months such as name, credit score, grography, age, tenure, balance, numOfProducts, credit card, active member, estimated salary, exited, etc. We also know which of these customers stayed and left. Now, given a new customer profile, can we create a model out of this data that can predict whether the customer will leave or stay?

Generic problem: It is a classification problem with lots of independent variables. Based on these variables we're predicting a binary outcome.

This problem is valuable to any customer-oriented organisations, e.g. should this person get a loan or not? is this a fradulent transaction?

def plot_distribution(dataset, cols=5, width=20, height=15, hspace=0.2, wspace=0.5):
    plt.style.use('seaborn-whitegrid')
    fig = plt.figure(figsize=(width,height))
    fig.subplots_adjust(left=None, bottom=None, right=None, top=None, wspace=wspace, hspace=hspace)
    rows = math.ceil(float(dataset.shape[1]) / cols)
    for i, column in enumerate(dataset.columns):
        ax = fig.add_subplot(rows, cols, i + 1)
        ax.set_title(column)
        if dataset.dtypes[column] == np.object:
            g = sns.countplot(y=column, data=dataset)
            substrings = [s.get_text()[:18] for s in g.get_yticklabels()]
            g.set(yticklabels=substrings)
            plt.xticks(rotation=25)
        else:
            g = sns.distplot(dataset[column])
            plt.xticks(rotation=25)

# Function that runs the requested algorithm and returns the accuracy metrics
def fit_algo(algo, X_train, y_train, X_test, cv):
    model = algo.fit(X_train, y_train)
    test_pred = model.predict(X_test)
    probs = model.predict_proba(X_test)[:,1]

    acc = round(model.score(X_test, y_test) * 100, 2) 
    # CV 
    train_pred = model_selection.cross_val_predict(algo, 
                                                  X_train, 
                                                  y_train, 
                                                  cv=cv, 
                                                  n_jobs = -1)
    acc_cv = round(metrics.accuracy_score(y_train, train_pred) * 100, 2)
    return train_pred, test_pred, acc, acc_cv, probs

# Part 2 - Making the ANN/MLP

# Importing the Keras libraries and package
# Sequential module - initialize neural network
# Dense - layers of ANN
import keras
from keras.models import Sequential
from keras.layers import Dense
from keras.layers import Dropout
# Initialising the ANN
classifier = Sequential()

# Adding the input layer and the first hidden layer with dropout
# Take average of input + output for units/output_dim param in Dense
# input_dim is necessary for the first layer as it was just initialized
classifier.add(Dense(6, input_dim = 11, kernel_initializer = 'glorot_uniform', activation = 'relu' ))

# Adding the second hidden layer with dropout
# doesn't need the input_dim params
# kernel_initializer updates weights
# activation function - rectifier
classifier.add(Dense(6, kernel_initializer = 'glorot_uniform', activation = 'relu' ))

# Adding the output layer
# dependent variable with more than two categories (3), output_dim needs to change (e.g. 3), activation function - sufmax
classifier.add(Dense(1, kernel_initializer = 'glorot_uniform', activation = 'sigmoid' ))

# Compiling the ANN - applying Stochastic Gradient Descent to whole ANN
# Several different SGD algorithms
# mathematical details based on the loss function
# binary_crossentropy, categorical_cross_entropy
classifier.compile(optimizer = 'adam', loss = 'binary_crossentropy', metrics = ['accuracy'])

# Fitting the ANN to the Training Set
# X_train, y_train, Batch size, Epochs (whole training set)
history=classifier.fit(X_train, y_train, batch_size = 16, epochs = 25, validation_data=(X_test, y_test))

Traditional Machine Learning¶

  • ### Algorithms: Logistic regression, SVM, decision trees
  • ### Characteristics
    • ### choice of algorithm
    • ### feature engineering
    • ### structured data (generated by systems of records)

Machine learning is about learning a problem specific function with a set of weights/parameters.

Machine learning needs feature engineering and careful selection of algorithms. The algorithms are shallow networks such as SVM or logistic regression.

Applications¶

  • Gmail: is this email a spam?

  • LinkedIn: is job J a job recommendation for user U?

  • Flipkart: is this a fraudulent transaction?

  • Bing: what is the relevance of Page P to query Q?

Deep Learning¶

  • ### Architectures: ConvNets, LSTM
  • ### Characteristics:
    • ### choice of architecture
    • ### automatic feature engineering
    • ### structured as well as unstructured data (text, images)
    • ### increase depth to learn complex functions

Deep learning is also about learning a function defined by an architecture and a set of weights/parameters. However, we don't need explicit feature engineering, and deep learning can perform better with increase in data and increase in depth of the network.

Applications¶

  • Google: is this image a spam?

  • Skype: translate english to hindi

  • Flipkart: show similar fashion products

  • Uber: should the car accelerate or stop?

A Single Layer Network is A Universal Approximator¶

If f(x) is a non-linear function, then in theory, a neural network even with 1 hidden layer can perfectly learn to solve any classification problem. That is, a neural network with a set of weights exists that can produce the classification targets from the input data.

Why do we then consider deep neural networks?

Shallow Network vs Deep Network¶

  • Long breadth vs tall depth (Shallow vs Deep neural networks)
  • Deep networks converge faster
  • Deep networks needs substantially lesser number of weights than shallow networks
  • Deep networks can learns hierarchical representation
  • Challenge: finding the right deep network architecture for a given problem
  • That is why we have variations such as CNN, LSTM with varying depths, varying activation functions, and different types of connections

Applications of deep learning¶

  • Image classification, object extraction from images and videos
  • Speech recognition and translation
  • Sentiment identification from text paragraphs

# overfitting is when the model has very high accuracy on training set but low accuracy on test set.
# this implies that the model does not generalize well, and does not perform well in practice.
# drop out is a regularization technique that sets random neurons to zero.
# this loosely models forgetting function in our brain
# this ensures that, over time, only strong neurons get fired and contribute to the model predictions
# weak neurons get turned off, and does not contribute to memorization/overfitting.

from keras.wrappers.scikit_learn import KerasClassifier
from sklearn.model_selection import GridSearchCV
from keras.models import Sequential
from keras.layers import Dense
def build_classifier(optimzer):
    classifier = Sequential()
    classifier.add(Dense(6, input_dim = 11, kernel_initializer = 'glorot_uniform', activation = 'relu' ))
    classifier.add(Dropout(p = 0.2))
    classifier.add(Dense(6, kernel_initializer = 'glorot_uniform', activation = 'relu' ))
    classifier.add(Dropout(p = 0.2))
    classifier.add(Dense(1, kernel_initializer = 'glorot_uniform', activation = 'sigmoid' ))
    classifier.add(Dropout(p = 0.2))
    classifier.compile(optimizer = optimzer, loss = 'binary_crossentropy', metrics = ['accuracy'])
    return classifier
classifier = KerasClassifier(build_fn = build_classifier)
parameters = {'batch_size': [25, 32],
              'nb_epoch': [100, 500],
              'optimzer': ['adam', 'rmsprop']}
grid_search = GridSearchCV(estimator = classifier,
                           param_grid = parameters,
                           scoring = 'accuracy', 
                           cv = 10)
grid_search = grid_search.fit(X_train, y_train)
best_parameters = grid_search.best_params_
best_accuracy = grid_search.best_score_
